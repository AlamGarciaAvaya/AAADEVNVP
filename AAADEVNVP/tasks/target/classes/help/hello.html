Notificación Vencimiento de Pago

Descripción de la tarea

La tarea se usa principalmente para recibir multiple strings o multiple numbers de una lectura de base de datos, cada inputParam se genera poniendo un "?" para Strings y un "°" para numbers en el cuerpo del mensaje. el caracter especial será sustituido por el valor asignado en el inputParam establecido para formar boceto de mensaje variando las especicifaciones por la posioción del caracter introducido.

Propiedades de la tarea

  • Body - Espacio multitext donde se introduce el mensaje que desea ser enviado, y se introducen los careacteres especiales "?" y "°" para crear inputParams en inputMapping
  • Email - Combo Box para elegir entre dos opciones, "Yes" para enviar por correo el mensaje escrito en Body al seleccionar esta opción se crea en el input Mapping un parámetro con nombre inputEmail el cual debe ser conectado a una variable multipleString con correos electronicos y "No" para no enviar el mensaje
  • SMS - Combo Box para elegir entre dos opciones, "Yes" para enviar por SMS el mensaje escrito en Body al seleccionar esta opción se crea en el input Mapping un parámetro con nombre inputSMS el cual debe ser conectado a una variable multipleString con una lista de numeros telefónicos moviles, y "No" para no enviar el mensaje
  • Input schema Show - Check box habilita multitext donde se mostrarán los paraámetros creados en el inputMapping
  • Retrieve - Botón que se usa para actualizar el registro de inputParams creados en el cuerpo del mensaje así como las opciones de Email y SMS

Input schema

  • inputParam (n) - Input creado dinámicamente de acuerdo a la posición de los caracteres especiales "?" y "°" introducidos en el Body
  • inputEmail - Input creado dinámicamente al momento que se selecciona la opción "Yes" en el combo box de Email, es necesario conectar el inputEmail a una variable multipleStrings que contenga cuentas de corre electrónico válidas.
  • inputSMS - Input creado dinámicamente al momento que se selecciona la opción "Yes" en el combo box de SMS, es necesario conectar el inputSMS a una variable multipleStrings que numeros telefónicos válidos.

Output schema

  • status - Indica si la operación seleccionada se ha realizado correctamente

Prerequisitos

Mapear una base de datos para obtener valores y conectar cada inputParam creado

Related Boundary Events

No aplica

Casos de uso

Crear mensajes con parametros dinámicos y enviar de manera masiva a diferenes usuarios.
